Diagrama de casos de uso

Comprar: Esta función permite a cualquier usuario comprar un artículo y pasar por la pasarela de pago.

Vender: Esta función permíte a cualquier usuario vender un artículo y recibirlo a través de distintos métodos bancarios.

Pujar: Esta función permíte a cualquier usuario pujar un artículo y en caso de ganarlo, pasar por la pasarela de pago.

Modificar: Esta función permite a cualquier usuario modificar cualquier anuncio o subasta.

Publicar: Esta función permite a cualquier usuario publicar un anuncio de cualquier producto.

Login: Esta función permite a cualquier usuario acceder a su cuenta.

Registro: Esta función permite a cualquier usuario registrarse en la aplicación.

Moderar interacciones: Esta función permite a cualquier administrador modificar, revisar o eliminar anuncios, usuarios o mensajes.

Subastar: Esta función permite a cualquier usuario crear una subasta y recibir la cuantía a través de distintos métodos bancarios.

Ejemplo de caso de uso

CU-03 Comprar producto
Precondiciones El sistema de gestión de MercaSwapp debe estar operativo. Al menos un producto con suficiente stock debe estar registrado en el catálogo
Descripción Permite a los clientes realizar una compra desde el catálogo, pudiendo seleccionar entre los distintos productos disponibles, variando el precio final.
Éxito El cliente realiza la compra de los productos seleccionados con éxito y se muestra la confirmación de la transacción
Fallo Se produce un error en el proceso de compra. La cantidad deseada de un producto no está y se produce una incidencia
Actor principal Cliente
Actor secundario Sistema
Qué datos usa Nombre y descripción del producto, así como el precio correspondiente dependiendo de la puja/subasta.
Secuencia normal
  • 1. El cliente accede al sistema de MercaSwapp
  • 2. El usuario selecciona la sección de “Clientes”
  • 3. El usuario presiona en el botón de “Comprar producto
  • 4. El sistema muestra el catálogo de productos, es decir, cada producto disponible con su nombre y descripción.
  • 5. El usuario selecciona el producto deseado.
  • 6. Presiona en el botón de “Agregar al carrito” y seguidamente, al de “Confirmar compra”
  • 7. Si el producto o los productos seleccionados han sido correctamente añadidos al carrito, entonces la compra se confirma y se añade a la base de datos correspondiente. En caso contrario, el sistema muestra un mensaje de que el carrito está vacío.
  • 8. Al confirmar la compra, el sistema registra la venta en la base de datos, en la tabla de ventas.
Excepciones Cuando el carrito de la compra está vacío, se muestra un mensaje y no se registra la venta.
Postcondiciones Se registra la venta y se actualiza el catalogo.
